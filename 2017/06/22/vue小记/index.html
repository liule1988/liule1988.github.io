#Vue #组件通信

最近一直在搞Vue开发，但是需求老是在变，一会儿要定制化的一面，一会儿又是要动态表单，搞得真的很头疼。一年了感觉自己还是只学到个前端的皮毛，难也！

不过这些天的研究对Vue也算越来越熟悉了。

今天就解决了之前的两个疑问。

  1. Vue父组件调用子组件定义的方法

    <Child ref='child'></Child>
<script>
    export default {
        name:'parent',
        methods:{
            showChildMethod(){
                this.$refs.child.test('我调用了子组件的方法哦');    //父调用子方法，并可传递参数。
            }
        }
    }

</script>

//子组件
<script>
    export default {
        name:'child',
        methods:{
            test(params){
                console.log(params)
            }
        }
    }

</script>

  2. Vue子组件改变父组件的方法（其实官方时不推荐这种方式，因为这违背了数据的单向流动原则，推荐用Vuex做状态管理。但是Vuex在表单绑定V-model会有问题。所以，有时候还是必须用到的，嘻嘻）

    <Child ref='child' @childToParent='change'></Child>
<script>
    export default {
        name:'parent',
        methods:{
            change(params){
                console.log('我收到了子组件的传值'+params)
                //可以在这里改变状态
            }
        }
    }

</script>

//子组件
<script>
    export default {
        name:'child',
        methods:{
            test(params){
                this.$emit('childToParent',params)  //向父组件通信
            }
        }
    }

</script>
